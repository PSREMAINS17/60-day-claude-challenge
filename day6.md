What changed and why it raised the score:

Rewrote the Summary with hard keywords — Added "MERN stack," "REST APIs," "full-stack," "scalable," and "LeetCode" — terms ATS parsers look for in software dev roles; the original was vague.
Standardized section headings — Changed to exact ATS-recognized labels: PROFESSIONAL SUMMARY, EDUCATION, PROJECTS, TECHNICAL SKILLS, CERTIFICATIONS — many parsers fail on custom or decorative headers.
Eliminated multi-column / icon layout — The original PDF used a two-column visual design that causes ATS parsers to scramble text order or skip content entirely; now fully single-column.
Expanded skill labels — Added "Object-Oriented Programming" alongside "OOPs" and "Computer Networks" alongside "CN" so both long-form and abbreviated keyword matches fire.
Strengthened project bullet action verbs — Replaced weak openers ("Developed," "Integrated" used repetitively) with varied, impact-forward verbs: "Built," "Architected," "Consumed," "Implemented," "Collaborated" — recruiters and parsers reward specificity.
Added technology context to bullets — Explicitly named Node.js, Express.js, ML models, and REST APIs inside project descriptions so keywords appear in context, not just the skills list.
Unified date format — Standardized all dates to Mon YYYY (e.g., Jan 2026 instead of Jan '26) — some ATS tools misparse abbreviated years.
Spelled out degree field — "B.Tech in Information Technology" instead of "B.Tech in IT" so the field name matches job description keyword scans for "Information Technology."
